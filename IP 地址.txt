	console.log(i)
	var b = parseInt(i).toString(2)
	var byte = Math.ceil(b.length / 8)
	return b.padStart(8*byte,'0').replace(/(\d{8})(?!$)/g, '$1 ')
}
var ipToB = s => {
	return s.replace(/(\/\d+)/g,'').split('.').map(toB).join(' ')
}

// DHCP（Dynamic Host Configure Protocol，动态主机配置协议）
DHCP服务器提供了三种IP分配方式：
	自动分配 (Automatic allocation)
	手动分配
	动态分配 (Dynamic Allocation)
	
自动分配是当DHCP客户端第一次成功的从DHCP服务器获取一个IP地址后，就永久的使用这个IP地址。

手动分配是由DHCP服务器管理员专门指定的IP地址

动态分配是当客户端第一次从DHCP服务器获取到IP地址后，并非永久使用该地址，每次使用完后，DHCP客户端就需要释放这个IP，供其他客户端使用。

IP地址 划分块写法
192.168.1.64/28
2^(32-28)=16
也就是以16为子网划分的块，这样一来，子网地址依次就是0、16、32、48、64等，也就是子网地址就是以16为基数的倍数。
不同块也就意味著子网所处的网络不同，也就是不在同一个网段里面，是无法进行通信的。

借用1位主机位，产生2个子网，每个子网有126个主机地址；借用2位主机位，产生4个子网，每个子网有62个主机地址。

每个子网中，第一个IP地址（即主机部分全部为0的IP）和最后一个IP（即主机部分全部为1的IP）不能分配给主机使用，所以每个子网的可用IP地址数为总IP地址数量"减2"；
